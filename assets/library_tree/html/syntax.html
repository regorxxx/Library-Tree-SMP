Notes:

  • Supports all TF expressions natively found on foobar2000:
    • Foobar2000 Title Formatting reference: https://wiki.hydrogenaudio.org/index.php?title=Foobar2000:Title_Formatting_Reference
    • Foobar2000 Album List reference: https://wiki.hydrogenaudio.org/index.php?title=Foobar2000:Titleformat_Album_List
  • Sorting is also based on TF output:
    • Alphanumerical sorting is done following the settings at 'Behaviour' tab (HTML options panel) or 'View By Sorting' (properties panel).
    • To sort by date, in most cases is preferable to use the 'Settings\Views\Sort selected view by' menu instead of directly adding an expression to the View pattern.
      This way you can easily change and switch sorting at any point without further editing.
    • 'View by Folder Structure' may also use its own independent windows-like sorting.
    • Playback Queue, Auto-DJ and Playlist sources may also use sorting by idx, set at 'Settings\Views'.
    • Sorting may be overridden by Search box or Filter expressions in some cases.
    • Panel Sorting priority: Search box > Filter > Source > View
  • Prefixes used by $stripbranchprefix{}, $swapbranchprefix{} and $prefix are set at advanced tab (HTML options panel) or 'Prefixes to Strip or Swap (| Separator)' (properties panel).
  • Rand functions are evaluated once for the entire view (TF expression), not per node. This is relevant for color usage for ex, since the same value will be applied across all the list.
  • Hard splitters (|) and Soft splitters (¦) don't filter empty valued tags by default, thus creating blank or '?' nodes.
    • Expressions like '%TAG%¦%FIELD%' will show a '?' node if %FIELD% does not exist, unless you use something like '%TAG%[¦%FIELD%]'.
    • Similarly, '%TAG%¦$meta(FIELD)' expressions should be replaced with '%TAG%[¦$meta(FIELD)]' to avoid blank nodes.
    • '$iterate{'$meta(FIELD,$counter)',2,¦}' expressions behave the same and they should be replaced with '$iterate{'[$meta(FIELD,$counter)]',X,¦}' which are then expanded to [$meta(FIELD,0)][¦$meta(FIELD,1)][¦...]
  • Support of $nowplaying{}, $selected{} and $nowplayingorselected{} is limited to display purposes.
    • May have some performance impact.
    • View will be updated automatically on Playback/selection changes.
    • Can be used for things like highlighting currently playing artist.
    • Don't try to use them for filtering purposes, use filters instead.
  • Usage of $sourcetype, $viewname, $filtername, etc. and can be used for conditional behavior.
    • May have some performance impact.
    • View will be updated automatically when filter/source changes.
    • Can be used for things like showing/hiding tags on specific filters or sources.
    • Don't try to use them for filtering purposes, use filters instead.

Notes:

  • Supports all TF expressions and Query syntax natively found on foobar2000:
    • Foobar2000 Title Formatting reference: https://wiki.hydrogenaudio.org/index.php?title=Foobar2000:Title_Formatting_Reference
    • Foobar2000 Query Syntax reference: https://wiki.hydrogenaudio.org/index.php?title=Foobar2000:Query_syntax
    • SORT BY expressions are also allowed (Filters and Search box):
      • Ascending: [...QUERY...] SORT BY [...]
      • Ascending: [...QUERY...] SORT ASCENDING BY [...]
      • Descending: [...QUERY...] SORT DESCENDING BY [...]
      • To apply sorting without filtering: ALL SORT BY [...]
      • Panel also allows to skip the query term: SORT BY [...]
      • Will override any View (TF) or Source sorting (queue idx, etc.) when used.
      • Panel Sorting priority: Search box > Filter > Source > View
  • Search box also supports RegExp. They must be written in /[expression]/[flags] form, where flags can be any of 'gimsuyt'.
    • Tags used for lookup are those present on the current View TF expression, ignoring those related to track and disc number; on folder view, paths are used instead.
    • RegExps and queries can not be mixed.
    • Non-official flag 't' applies transliteration to tag values before matching.
      Transliteration supports Cyrillic, Greek, Chinese (Pinyin) and Japanese (Katakana and Hiragana), like quick-search and sorting.
    • RegExp reference: https://regexr.com/
  • Search box expression is applied on top of Filters. i.e. after current View and Filter patterns have been processed.
  • Search box expression may be updated on playback or selection changes. i.e. functionally equivalent to Filters.
    • Great performance impact, when used, compared to Filters.
    • It's preferable to directly use a Filter than permanently using the search box with a dynamic expression to do the same. Filters are much faster.
    • Playback/selection changes automatic evaluation can be switched at 'Search' tab (HTML options panel) or 'Search Auto-refresh TF Expressions' (properties panel).
    • If automatic evaluation is disabled, then search can be refreshed by pressing 'enter'.
  • Usage of $sourcetype, $viewname, $filtername, etc. and can be used for conditional behavior.
    • May have some performance impact.
    • Filter will be updated automatically when view/source changes.
    • Can be used for things like showing/hiding tracks on specific views or sources.
    • It's preferable to directly use a Filter than permanently using the search box with these variables to do the same. Filters are much faster.

Notes:

  • Supports all TF expressions natively found on foobar2000:
    • Foobar2000 Title Formatting reference: https://wiki.hydrogenaudio.org/index.php?title=Foobar2000:Title_Formatting_Reference
  • Note Playlist/Selection sorting is independent to Panel sorting (which is only used if not specified).
  • Special sorting functions can be applied on top of any standard TF expressions, being replaced with $not(0) on real time. The sorting algorithms will then be applied before the standard TF sorting.
  • Execution order: harmonicsort -> harmonicmix -> shufflebytags -> standard TF
  • Conditional execution can be partially achieved, but note the special sorting functions are evaluated only once (for all tracks). This means that TF expressions based on tags will probably not work as expected except when evaluating against playback or selection. For example:

    • $if($strcmp(%ARTIST%,$nowplaying{%ARTIST%}),0,1)$harmonicmix{}
      Always does harmonic mixing, then puts now playing artist first, leaving the rest as previously sorted.

    • $if($strcmp(%ARTIST%,$nowplaying{%ARTIST%}),0$harmonicmix{},1)
      Only does harmonic mixing while playing in which case also puts now playing artist first, leaving the rest as previously sorted. When not playing, leaves all tracks as selected from panel.
      Note this is pretty different to only applying harmonic mixing when tracks' artists match the now playing artist.

    • $if($strcmp(%ARTIST%,$nowplaying{%ARTIST%}),0,1$harmonicmix{})
      Only does harmonic mixing while not playing. When playing, puts now playing artist first and leaves all other tracks as selected from panel.
      Note this is pretty different to only applying harmonic mixing when tracks' artists don't match the now playing artist.

    • $if($strcmp(%ARTIST%,$nowplaying{$replace(%ARTIST%,',',''',''')}),0,1$harmonicmix{})
      Instead of the previous examples, the nowplaying part should be sanitized so multi-value tags works by escaping ','.

    • $if($nowplaying{%ARTIST%},$harmonicmix{})
      Will never do harminic mixing, since the evaluated nowplaying part, being a string in any case, is treated as false.
  • For conditional execution based on player state is better to use global variables like '%ISPLAYING%'', '%ISPAUSED%', ... For example:

    • $if(%ISPLAYING%,$harmonicmix{})
      Only does harmonic mixing while playing.

    • $if(%ISPLAYING%,$if($strcmp(%ARTIST%,$nowplaying{%ARTIST%}),0,1))
      While playing puts the now playing artist first and leaves all other tracks as selected from panel.

    • $if(%ISPLAYING%,$harmonicmix{}$if($strcmp(%ARTIST%,$nowplaying{%ARTIST%}),0,1))
      Only does harmonic mixing while playing and also puts the now playing artist first, leaving the rest as previously sorted.
  • Conditional execution based on panel settings like Source, View or Filter works as expected:

    • $if($strcmp($sourcetype,library),$harmonicmix{},)
      Only does harmonic mixing on 'Library' source and sorts all other cases as selected from panel.

    • $if($strcmp($viewname,test),$harmonicmix{},)
      Only does harmonic mixing on 'Test' view and sorts all other cases as selected from panel.

    • $if($strcmp($filter,lossless),,$harmonicmix{})
      Only does harmonic mixing when filter is not 'Lossless' and sorts while using 'Lossless' filter as selected from panel.
